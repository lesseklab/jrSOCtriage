jrSOCtriage - Useful Graylog Searches
======================================

All searches should be run against the jrsoc stream or All Messages
with an appropriate time range. Fields use double quotes for string values.

NOTE: Graylog 5 requires string fields to be quoted.
      Boolean-like fields (true/false) must be quoted: "true" not true.
      Leading wildcards (*) are not supported in WildcardQuery.

============================================================
DEPLOYMENT / ORG FILTERING
============================================================

On a shared Graylog receiving from multiple jrSOCtriage instances,
scope to one organization or security domain. Set deployment.org and
deployment.security_domain in each instance's config (Config tab,
Deployment card); they ship as jrsoc_org and jrsoc_security_domain on
every record. Combine these with any search below to limit results.

All alerts from one organization:
  jrsoc_org:"acme-corp"

All alerts from one security domain within an org:
  jrsoc_org:"acme-corp" AND jrsoc_security_domain:"production"

NOTIFY verdicts for one org (combine with a status search):
  jrsoc_org:"acme-corp" AND gl2_llm_verdict:NOTIFY

Records missing a deployment label (instances not yet configured;
unset fields ship as "N/A"):
  jrsoc_org:"N/A"

============================================================
PIPELINE STATUS SEARCHES
============================================================

All enriched alerts (shipped by jrSOCtriage, not yet triaged):
  gl2_triage_complete:"false"

All triaged alerts (reviewed by LLM):
  gl2_triage_complete:"true"

All NOTIFY verdicts:
  gl2_llm_verdict:NOTIFY

All NOTE verdicts (use as saved search for periodic review instead of email):
  gl2_llm_verdict:NOTE

All SUPPRESS verdicts:
  gl2_llm_verdict:SUPPRESS

All FAILED verdicts (pipeline could not reach the LLM — endpoint issue):
  gl2_llm_verdict:FAILED

Pipeline health check (count of FAILED in last hour — good candidate for
a Graylog alert if threshold exceeded):
  gl2_llm_verdict:FAILED AND gl2_llm_endpoint:pipeline

High confidence NOTIFY:
  gl2_llm_verdict:NOTIFY AND gl2_llm_confidence:HIGH

Medium confidence NOTIFY:
  gl2_llm_verdict:NOTIFY AND gl2_llm_confidence:MEDIUM

Alerts with LLM prompt stored (for prompt review/testing):
  _exists_:gl2_llm_prompt AND gl2_triage_complete:"true"

Alerts with ntopng flow data:
  _exists_:gl2_ntopng_flows

Alerts with Graylog context logs:
  _exists_:context_logs

============================================================
ESCALATION AND ENRICHMENT SEARCHES
============================================================

Alerts escalated by abuse score:
  gl2_abuse_score:>50

Alerts with external IPs (enriched):
  _exists_:gl2_src_country

Alerts from a specific country:
  gl2_src_country:"Russia"

Alerts from known scanner infrastructure (examples):
  gl2_src_rdns:*shodan*
  gl2_src_rdns:*censys*
  gl2_src_rdns:*stretchoid*

Alerts above hourly baseline:
  gl2_baseline_note:*ABOVE HOURLY*

Alerts above daily baseline:
  gl2_baseline_note:*ABOVE DAILY*

============================================================
THREAT INTELLIGENCE SEARCHES (v1.1)
============================================================

Five per-indicator intelligence sources ship flat fields for searching.
Two field behaviors, and the difference matters:

  Always-present strings ("N/A" when nothing applies):
    gl2_kev_listed, gl2_greynoise_class
  Numeric, present only when something was checked:
    gl2_epss_max, gl2_vt_malicious, gl2_otx_pulses,
    gl2_mentioned_abuse_max, gl2_mentioned_vt_malicious,
    gl2_mentioned_otx_pulses
    Absent means UNCHECKED (source disabled, no qualifying indicator,
    or lookup unavailable). 0 means CHECKED and nothing found — a real
    answer. Use _exists_ to tell coverage apart from results.
  Present-only string:
    gl2_mentioned_greynoise_worst (absent when no mentioned external
    IP produced a real answer)

--- CISA KEV (actively-exploited CVEs) ---

Alert carries a CVE confirmed exploited in the wild:
  gl2_kev_listed:"true"

CVE present but not in the KEV catalog:
  gl2_kev_listed:"false"

Verdicts made without KEV context (catalog degraded — worth review):
  gl2_kev_listed:STALE OR gl2_kev_listed:UNAVAILABLE

--- EPSS (exploitation probability) ---

Any CVE scored:
  _exists_:gl2_epss_max

High exploitation probability (30-day, modeled):
  gl2_epss_max:>0.5

Not yet KEV-listed but likely to be exploited (the forward-looking pair):
  gl2_kev_listed:"false" AND gl2_epss_max:>0.5

--- GreyNoise (internet noise vs. targeted) ---

Source IP NOT seen mass-scanning — the targeted-activity signal:
  gl2_greynoise_class:not_seen

Known-benign infrastructure (common-business services or benign scanners):
  gl2_greynoise_class:riot OR gl2_greynoise_class:benign

Known-malicious mass scanner:
  gl2_greynoise_class:malicious

Lookups skipped by rate limit (free-tier quota audit):
  gl2_greynoise_class:RATE_LIMITED

Coverage check — external-IP alerts that actually got a GreyNoise answer:
  _exists_:gl2_greynoise_class AND NOT gl2_greynoise_class:"N\/A"
  (the / in N/A needs escaping in Graylog query syntax)

--- VirusTotal (antivirus engine detections) ---

Any indicator flagged malicious by at least one engine:
  gl2_vt_malicious:>0

Strong multi-engine consensus:
  gl2_vt_malicious:>5

Checked and clean (all indicators, zero engines):
  gl2_vt_malicious:0

--- AlienVault OTX (community threat reports) ---

Indicator referenced in community pulses:
  gl2_otx_pulses:>0

Checked, no community reports:
  gl2_otx_pulses:0
  (Field value 50 means 50-or-more — the count saturates.)

--- Mentioned-IP intel (IPs in alert text, not parties) ---

The per-source fields above describe IPs that are PARTIES to the
alert (structured source/destination fields). External IPs that
merely appear in the alert's raw log text — DNS answers on a domain
controller, proxied addresses, log references — ship separate
worst-of fields instead, so a DC alert whose log contains a resolved
malicious IP never pollutes the party fields:

Alert text references a known-malicious mass scanner:
  gl2_mentioned_greynoise_worst:malicious

Alert text references high-abuse-score infrastructure:
  gl2_mentioned_abuse_max:>50

VT/OTX on mentioned IPs are HASH-GATED: they run only when the
alert also carries a hash change (IP reputation corroborating hash
intel on one record); hash-less alerts spend no VT/OTX quota on
mentioned IPs, so these two fields exist only on hash-bearing
alerts:

Alert text references VT-flagged infrastructure:
  gl2_mentioned_vt_malicious:>0

Alert text references community-reported infrastructure:
  gl2_mentioned_otx_pulses:>0

Different question from the party fields: "the alert TEXT references
known-bad infrastructure" vs "the alert's counterparty is known-bad."

--- Combined ---

Engine detections AND community reporting on the same alert:
  gl2_vt_malicious:>0 AND gl2_otx_pulses:>0

Actively-exploited CVE that the LLM escalated:
  gl2_kev_listed:"true" AND gl2_llm_verdict:NOTIFY

  wazuh_rule_id:RULE_ID


============================================================
ANONYMIZATION AUDIT SEARCHES
============================================================

All alerts sent to cloud LLM endpoints with anonymization:
  gl2_anon:"true"

All alerts sent to local LLM endpoints (not anonymized):
  gl2_anon:"false" AND gl2_triage_complete:"true"

Anonymized alerts by endpoint:
  gl2_anon:"true" AND gl2_llm_endpoint:gemini-2.5-flash
  gl2_anon:"true" AND gl2_llm_endpoint:gpt-4o

All cloud-triaged NOTIFY verdicts (anonymized prompts):
  gl2_anon:"true" AND gl2_llm_verdict:NOTIFY

Compare local vs cloud verdict distribution:
  gl2_anon:"false" AND gl2_llm_verdict:NOTIFY
  gl2_anon:"true" AND gl2_llm_verdict:NOTIFY

Verify anonymization is working — confirm no real
hostnames/usernames appear in cloud-endpoint prompts:
  gl2_anon:"true" AND _exists_:gl2_llm_prompt
  (review gl2_llm_prompt field manually for any PII leakage)

============================================================
COMBINED SEARCHES (most useful for daily review)
============================================================

LLM triaged alerts in last hour:
  gl2_triage_complete:"true"
  (set time range to Last 1 hour)

External threats to a specific host:
  canonical_hostname:HOSTNAME AND gl2_triage_complete:"true"

High abuse score alerts that went to LLM:
  gl2_abuse_score:>50 AND gl2_triage_complete:"true"

All NOTIFYs with reasoning:
  gl2_llm_verdict:NOTIFY AND _exists_:gl2_llm_reasoning

Alerts with both ntopng and LLM data:
  _exists_:gl2_ntopng_flows AND gl2_triage_complete:"true"

============================================================
SAVED SEARCH RECOMMENDATIONS
============================================================

Consider saving these as named searches in Graylog for quick access:

"LLM Review"         gl2_triage_complete:"true"
                     Columns: timestamp, source, gl2_llm_verdict,
                     gl2_llm_confidence, message, gl2_mitre_tactics

"NOTIFY Alerts"      gl2_llm_verdict:NOTIFY
                     Columns: timestamp, source, gl2_llm_verdict,
                     gl2_llm_confidence, gl2_llm_summary, gl2_mitre_tactics

"Notes Review"       gl2_llm_verdict:NOTE
                     Columns: timestamp, source, gl2_llm_confidence,
                     gl2_llm_summary, gl2_llm_reasoning
                     Check periodically instead of using email for NOTEs

"External Threats"   _exists_:gl2_src_country AND gl2_triage_complete:"true"
                     Columns: timestamp, source, gl2_src_rdns,
                     gl2_src_country, gl2_abuse_score, gl2_llm_verdict

"DMZ Activity"       canonical_hostname:YOUR_DMZ_HOST
                     Columns: timestamp, gl2_src_rdns, gl2_src_country,
                     gl2_abuse_score, gl2_llm_verdict, message

"Cloud Audit"        gl2_anon:"true"
                     Columns: timestamp, source, gl2_llm_endpoint,
                     gl2_llm_verdict, gl2_llm_confidence, gl2_anon

"Threat Intel Hits"  gl2_kev_listed:"true" OR gl2_vt_malicious:>0 OR
                     gl2_otx_pulses:>0 OR gl2_greynoise_class:malicious
                     Columns: timestamp, source, gl2_kev_listed,
                     gl2_epss_max, gl2_vt_malicious, gl2_otx_pulses,
                     gl2_greynoise_class, gl2_llm_verdict
